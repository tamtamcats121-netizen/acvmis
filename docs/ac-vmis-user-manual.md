# AC-VMIS User Manual

## 1. Introduction

The Asian College Varsity Management Information System, or AC-VMIS, is a web-based system used to manage varsity student-athletes, coaches, teams, schedules, attendance, academic eligibility, documents, announcements, and reports.

This user manual explains how each user role uses the system. It is arranged in the recommended order of use, starting from account access and ending with reports, support, and troubleshooting.

## 2. System Users

AC-VMIS has three main user roles:

- `Administrator`: monitors system records, users, teams, documents, academics, attendance, reports, and audit logs.
- `Coach`: reviews student-athlete applications, manages teams, schedules, attendance, and team documents.
- `Student-Athlete`: views team information, schedules, academic status, submitted documents, and announcements.

## 3. Getting Started

### 3.1 Accessing the System

Users access AC-VMIS through the system website using a browser. The user must have an active account before accessing role-based pages.

### 3.2 Logging In

1. Open the AC-VMIS login page.
2. Enter the registered email address.
3. Enter the account password.
4. Select `Sign In`.
5. The system redirects the user to the correct dashboard based on role.

### 3.3 Account Status Pages

The system may redirect users to a status page depending on account condition:

- `Pending Approval`: The student-athlete registration is waiting for coach review.
- `Rejected`: The student-athlete registration was not approved.
- `Deactivated`: The account was temporarily disabled.

### 3.4 Password Reset

1. Open the forgot password page.
2. Enter the registered email address.
3. Submit the password reset request.
4. Follow the reset link sent by email.
5. Enter and confirm the new password.

## 4. Student-Athlete User Manual

### 4.1 Registration

1. Open the registration page.
2. Enter personal and account information.
3. Select the sport being applied for.
4. Upload required registration documents, if requested.
5. Submit the registration form.
6. Wait for coach approval.

After submission, the account remains pending until the coach assigned to the selected sport approves or rejects the application.

### 4.2 Dashboard

The Student-Athlete Dashboard shows important reminders, account status, team assignment, schedule information, academic alerts, and announcements.

Use this page to quickly check whether any action is needed.

### 4.3 Team Page

The Team page shows the assigned varsity team.

Information shown may include:

- Team name
- Sport
- Coach information
- Assistant coach information
- Player list
- Jersey number
- Position

### 4.4 Joining a Team by Invite Code

1. Open the team invite link or go to the join team page.
2. Enter the team invite code.
3. Submit the code.
4. The system checks account approval, sport match, roster availability, and team status.
5. If valid, the student-athlete is added to the team roster.

### 4.5 Schedule

The Schedule page shows team practices, games, and other team activities.

Use this page to check:

- Schedule title
- Type of activity
- Venue
- Date and time
- Attendance or participation requirements
- Training requirements, if assigned

### 4.6 Academics

The Academics page is used to submit academic requirements and view eligibility results.

To submit an academic document:

1. Open `Academics`.
2. Select the active academic period.
3. Upload the grade report or required academic file.
4. Submit the file.
5. Wait for scanning, validation, and evaluation.

Possible academic results:

- `Eligible`: Varsity access is allowed.
- `Pending Review`: The submission still needs review or final evaluation.
- `Ineligible`: Varsity features may be temporarily limited.

If the student-athlete is ineligible or pending review, the system may restrict access to dashboard, team, and schedule pages. Academics and document submission remain available so the student can resolve requirements.

### 4.7 My Documents

The My Documents page shows documents submitted by the student-athlete.

Available actions:

- `Preview`: Opens the uploaded document.
- `Download`: Downloads the uploaded document.

### 4.8 Help and Support

The Help and Support page gives role-based guidance and common answers. Students should include their full name, student number, team, and issue details when asking for help.

## 5. Coach User Manual

### 5.1 Coach Account Activation

Coach accounts are created by administrators. The coach receives an activation link and uses it to set a password.

Steps:

1. Open the activation link.
2. Enter the required account details.
3. Set a password.
4. Submit the activation form.
5. Log in using the activated account.

### 5.2 Coach Dashboard

The Coach Dashboard shows team-related summaries, schedule reminders, attendance reminders, academic alerts, and recent activity.

### 5.3 Student Applications

Coaches review student-athlete applications for their assigned sport.

To approve or reject an application:

1. Open `Student Applications`.
2. Review the student-athlete information.
3. Check submitted registration documents.
4. Select approve or reject.
5. Add remarks if needed.
6. Submit the decision.

Approved student-athletes become available for roster assignment under the matching sport.

### 5.4 Team Management

Coaches can create and manage teams for their assigned sport.

Team management includes:

- Creating a team
- Updating team details
- Assigning an assistant coach
- Adding approved student-athletes to the roster
- Updating player positions
- Archiving or reactivating a team

### 5.5 Team Invite Code

Coaches may generate a team invite code so eligible student-athletes can join a team.

Available invite code actions:

- `Generate Invite Code`: Creates a team code.
- `Copy Link`: Copies the invite link.
- `Copy Code`: Copies the code only.
- `View Join Page`: Opens the student join page.
- `Regenerate Code`: Creates a new code and invalidates the old one.
- `Disable Code`: Stops students from joining through the current code.

### 5.6 My Team

The My Team page shows team information, coach details, assistant coach details, invite code tools, and roster records.

Coaches can review:

- Player details
- Student ID
- Position
- Jersey number
- Player status
- Contact information
- Emergency contact information

### 5.7 Schedule and Attendance

Coaches use Schedule and Attendance to manage team sessions.

To create a schedule:

1. Open `Schedule and Attendance`.
2. Add a new schedule.
3. Enter title, type, venue, start time, end time, and notes.
4. Save the schedule.

To record attendance:

1. Open the schedule.
2. Open the attendance roster or attendance sheet.
3. Mark each student-athlete as present, absent, late, or excused.
4. Save the attendance record.

### 5.8 Training Requirements

Coaches may assign training requirements linked to schedules and student-athletes.

Training requirement details may include:

- Schedule
- Student-athlete
- Coach
- Title
- Category
- Description

### 5.9 Team Documents

The Team Documents page allows coaches to view documents submitted by assigned student-athletes.

Available actions:

- `Preview`: Opens the uploaded document.
- `Download`: Downloads the uploaded document.

## 6. Administrator User Manual

### 6.1 Admin Dashboard

The Admin Dashboard gives an overview of system activity, attendance, academic status, pending items, team activity, and recent records.

### 6.2 User Records

The User Records page is used to monitor accounts.

Administrators can:

- Search users
- Filter by role or status
- View user profile records
- Deactivate accounts
- Reactivate accounts
- Create coach accounts
- Send coach activation links
- Send admin invitations

### 6.3 Coach Account Management

To create a coach account:

1. Open `User Records`.
2. Select the option to create a coach account.
3. Enter coach information.
4. Assign a supported sport.
5. Create the account and send the activation link.

### 6.4 Team Monitoring

Administrators can monitor teams, rosters, coaches, archive status, and team records.

Admin team monitoring includes:

- Viewing active teams
- Viewing team roster
- Viewing assigned coaches
- Viewing assigned players
- Archiving teams
- Reactivating teams
- Printing rosters

### 6.5 Operations Workspace

The Operations workspace is used to monitor schedules and attendance records.

Administrators can:

- View schedules
- Review attendance records
- Check attendance exceptions
- Update attendance manually when authorized
- Print schedules
- Print attendance registers

### 6.6 Documents

The Documents page shows uploaded student documents.

Available actions:

- `Preview`: Opens the document.
- `Download`: Downloads the document.

### 6.7 Academics

The Academics module is used to manage academic periods, submissions, eligibility evaluations, and academic exceptions.

Administrators can:

- Create academic periods
- Open or close academic periods
- View active academic period details
- View Academic Period History
- Review academic submissions
- View eligibility evaluations
- Update evaluations when needed
- Delete documents when authorized

### 6.8 Reports

Administrators can generate reports for monitoring and documentation.

Available reports:

- Attendance Report
- Roster Report
- Academic Report

Reports may include print and export options depending on the report page.

### 6.9 Audit Trail

The Audit Trail records important system actions.

Administrators can use it to review:

- Student approval activity
- Account actions
- Academic evaluations
- System record changes

### 6.10 Announcements

Administrators can view announcements and system notifications. Announcements help users stay informed about approvals, academic submissions, schedule updates, and general system events.

## 7. Common System Functions

### 7.1 Previewing Files

Select `Preview` to open a submitted file in the browser.

### 7.2 Downloading Files

Select `Download` to save a copy of the submitted file.

### 7.3 Printing Records

Some pages include print options for rosters, attendance records, schedules, academic summaries, and reports.

### 7.4 Searching and Filtering

Many tables include search and filter controls. Users may filter records by status, date, type, period, team, or role depending on the page.

### 7.5 Notifications

Notifications appear when important events happen, such as account approval, academic submission, or schedule-related updates.

## 8. Troubleshooting

### 8.1 Cannot Log In

Possible reasons:

- The email or password is incorrect.
- The account is pending approval.
- The account was rejected.
- The account was deactivated.
- The user needs to reset the password.

### 8.2 Student Cannot Access Dashboard, Team, or Schedule

Possible reasons:

- The account is still pending approval.
- The student-athlete has no team assignment yet.
- The student-athlete is academically ineligible.
- The academic submission is still pending review.

### 8.3 Student Cannot Join a Team

Possible reasons:

- The invite code is incorrect.
- The invite code is disabled.
- The invite code expired.
- The student-athlete is not approved.
- The student-athlete selected a different sport.
- The student-athlete is already assigned to another active team.

### 8.4 Document Upload Failed

Possible reasons:

- The file type is not accepted.
- The file is too large.
- The internet connection was interrupted.
- The uploaded scan is unclear.
- The academic period is closed.

### 8.5 Attendance Record Is Incorrect

The user should report the issue with the schedule date, team name, student-athlete name, and the expected attendance status.
